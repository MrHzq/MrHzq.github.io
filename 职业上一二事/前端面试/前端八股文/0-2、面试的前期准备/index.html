0-2、面试的前期准备

本文最后更新于 2024-03-22T23:32:24+00:00

各面考察度

一面

考察：广度，范围为：基础知识点、实际操作(代码手写)、项目结合

基础知识点

  • 一面一般是业务组骨干，他关心的是你能不能胜任组里面的工作，知识点是否齐全，所以一面最难，因为要求最全。

实际操作指的是

  • 线上面试时的手写代码环节，线上更容易考察写代码的能力，所以面试题的手写一定要去加强。

项目结合考察的是

  • 你掌握的知识点是否自我消化，是否融入到项目中了
  • 这时候就要去结合第一点(基础知识)，每掌握一个知识，就反向看看自己的项目中是否使用过，有的话加强理解，没的话新建项目去使用与了解，这样知识点才能与项目串联起来

二面

考察：深度，深挖基础知识

三面

考察：创新度，结合业务/场景，使用技术解决问题的新颖度

如何复习

当学习一个新知识点时，会处于以下三个状态：

  • 吃不下
    • 这时候就要先补齐缺失的知识点，然后经典面试题一定要掌握
  • 刚刚好
    • 针对性补充，可以出去面试小厂了，以面代练
  • 不够吃
    • 已经准备的很充分了，那就要把知识结合到项目中，找自己的两点，这时候可以去面试大厂了

如何提升

自我提升思路：在业务之外去发现问题，并解决掉它

找寻突破

突破顺序：发现问题 => 找到问题 => 获取痛点 => 制定方案(调研行业方案？自研？原因？) => 解决问题

突破领域

从最近比较火/前沿/流行/新颖的领域下手
最近流行：

  1. 整合
    1. 技术设计和产品设计的整合
      1. 规范化
      2. 统一化
      3. 可视化
      4. 智能化
  2. 效率提升
  3. 性能优化
    1. 老的：工程化
    2. 新的：web 3.0、web 二进制
    3. 更新：整合式，同一套组件驱动页面

成长路线

简历书写意见

  1. 不要把项目说的很牛逼，因为项目牛逼不代表你个人的牛逼，要强调的是：个人在项目中把多少知识转化成对项目的贡献，这才是你牛逼的点
